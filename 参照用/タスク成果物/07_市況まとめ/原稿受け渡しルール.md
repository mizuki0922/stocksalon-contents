# 原稿受け渡しルール

> **目的**: 講座責任者と運営スタッフの間で、原稿のやり取りをスムーズに行う

---

## 基本情報

| 項目 | 内容 |
|------|------|
| 原稿作成者 | 講座責任者 |
| 受け取り・投稿担当 | 運営スタッフ（小谷） |
| 配信日 | 火曜・金曜 20:00 |

---

## 受け渡し方法（案）

以下のいずれかを講座責任者と決めてください。

### 案1: チャット（Slack / LINE / Discord DM）

| 項目 | 内容 |
|------|------|
| 方法 | チャットに原稿を直接貼り付け |
| メリット | 手軽、通知が来るので見逃しにくい |
| デメリット | 長文だと見づらい |
| 向いているケース | 原稿が短め、やり取りが頻繁 |

**運用イメージ:**
```
講座責任者：「本日分の原稿です」→ 原稿を貼り付け
運営スタッフ：「確認しました。OKです！」
```

---

### 案2: 共有フォルダ（Google Drive / Dropbox）

| 項目 | 内容 |
|------|------|
| 方法 | 共有フォルダに原稿ファイルを保存 |
| メリット | 過去の原稿が残る、整理しやすい |
| デメリット | フォルダを開く手間、通知がない |
| 向いているケース | 原稿を保存・管理したい |

**フォルダ構成の例:**
```
市況まとめ原稿/
├── 2026年3月/
│   ├── 0303_市況まとめ.txt
│   ├── 0306_市況まとめ.txt
│   └── ...
└── 2026年4月/
    └── ...
```

**運用イメージ:**
```
講座責任者：フォルダに原稿をアップ → チャットで「アップしました」
運営スタッフ：確認 → チャットで「OKです」
```

---

### 案3: メール

| 項目 | 内容 |
|------|------|
| 方法 | メールで原稿を送付 |
| メリット | 記録が残る、フォーマルな形式 |
| デメリット | 確認が遅れることがある |
| 向いているケース | 記録を重視したい |

---

## 推奨案

**チャット（案1）を推奨**

理由：
- 通知が来るので見逃しにくい
- 「OK」の返信もすぐできる
- 市況まとめは500〜1,000文字程度なので、チャットで十分

---

## 締切と流れ

### 通常配信（火曜・金曜）

| 時間 | 担当 | やること |
|------|------|----------|
| 〜18:00 | 講座責任者 | 原稿をチャットに送る |
| 18:00〜19:30 | 運営スタッフ | 内容を確認し「OK」を返す |
| 20:00 | 運営スタッフ | Discord #市況まとめ に投稿 |
| 20:00〜 | 運営スタッフ | 投稿完了を報告 |

### 原稿が届かない場合

| 時間 | やること |
|------|----------|
| 17:30 | まだ届いていなければ、講座責任者に確認連絡 |
| 18:30 | それでも届かない場合、本日の配信をスキップするか相談 |

---

## 確認時のやり取り例

### 原稿を受け取ったとき

```
原稿確認しました。問題ありません！
20:00に配信します。
```

### 修正が必要なとき

```
原稿確認しました。
以下の点をご確認いただけますか？

・〇〇の数字が△△になっていますが、正しいでしょうか？
・「必ず上がる」の表現は避けた方がよいので、「上昇が期待される」に変更してもよいですか？

修正版をお待ちしています。
```

### 投稿完了の報告

```
市況まとめ、配信完了しました！
```

---

## 臨時配信の場合

### 講座責任者からの連絡

```
【臨時配信依頼】
本日、日経平均が-3.5%と大きく下げたので、臨時で配信したいです。
原稿を30分後に送りますので、配信をお願いできますか？
```

### 運営スタッフの対応

```
承知しました！原稿お待ちしています。
届き次第、確認して配信します。
```
