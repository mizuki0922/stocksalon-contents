2.1特典コンテンツの作成（入会時提供分） 一番最優先 初めては難しめ最優先02_特典コンテンツ作成
やさしい言葉で小分け
  • ①銘柄の選び方の動画（講座責任者と）
    【初心者向けステップ】
    1. 講座責任者と「この動画で何を伝えるか」を話し合い、見出しと流れ（構成）を決めます。
    2. その構成に沿って、スライドの文と話す原稿を書きます。自分で書くか、講座責任者に書いてもらうか決めておきます。
    3. 書き終えたら、必ず講座責任者に内容を確認してもらいます。直すところがあればメモしてもらいます。
    4. 撮影の日時・場所を決め、カメラ・マイクを準備して撮影します。原稿を読み上げる形でOKです。
    5. 撮影した動画をつなぎ、不要な部分をカットし、必要なら字幕を入れます（編集ソフト or 技術担当に相談）。
    6. 編集が終わった動画を講座責任者に見せ、最終OKをもらってから公開します。順番を間違えるとやり直しになるので、確認→撮影→編集→最終確認の流れを守ってください。
  • ②財務の見方の動画（講座責任者と）
    【①銘柄の選び方の動画と同じ流れで進めます】
    1. 財務の見方で「伝えたいこと」を講座責任者と決め、見出しと流れ（構成）を書きます。
    2. 構成に沿って原稿とスライドを書きます。
    3. 講座責任者に内容を確認してもらい、直しを入れます。
    4. 撮影の日時・場所を決めて撮影し、編集（つなぎ・カット・字幕など）をします。
    5. 編集後の動画を講座責任者に渡し、最終OKをもらってから使います。途中で確認を飛ばさないことが大切です。
  • ③売買のタイミングの動画（講座責任者と）
    【①・②の動画と同じ手順です】
    1. 売買のタイミングで伝える内容を講座責任者と決め、構成（見出し・流れ）を書きます。
    2. 原稿・スライドを書き、講座責任者に確認してもらいます。
    3. 撮影→編集をし、最後に講座責任者に最終確認してもらってから完成にします。流れを守れば失敗しにくいです。
  • 銘柄チェックリストの文章を書く（講座責任者と）
    【ステップ】
    1. 銘柄を選ぶときに「何をチェックするか」を講座責任者と話し合い、項目をリストにします（例：業績・株価の動き・ニュースなど）。
    2. 講座責任者に各項目の説明や基準を書いてもらうか、一緒に文章を決めます。
    3. 書いた内容を一度読み返し、抜けや矛盾がないか確認します。分からない用語があれば講座責任者に聞いて、やさしい言葉に直しておきます。
  • チェックリストをシート（表）にする（講座責任者に最終確認）
    【ステップ】
    1. Excel または Google スプレッドシートを開き、新しいシートを作ります。
    2. 1行目に「列の名前」を書きます。例：銘柄名・チェック項目1・チェック項目2・メモ・日付 など。
    3. 2行目以降は、会員が記入する欄として空白にしておきます。
    4. 列の幅や見出しの色を整え、講座責任者に「この形でよいか」最終確認してもらいます。表の形を先に決めておくと、あとで混乱しません。
  • チェックリストの最終チェック（講座責任者）
    【ステップ】
    1. チェックリストの文章と表を、最初から最後まで自分で一度読みます。誤字・抜けがないかチェックします。
    2. 講座責任者に「内容と数字が正しいか」確認してもらいます。特に数値や専門用語は間違いやすいので、必ず見てもらいます。
    3. 直しがあれば反映し、もう一度確認してもらいます。
    4. 「OK」の返事をもらったら完了です。OKをもらうまでは会員に渡さないようにします。
  • メルマガ Day1〜30 の原稿を仕上げる（講座責任者と）
    【ステップ】
    1. まず Day1〜Day30 の「テーマ一覧」と「いつ配信するか」のスケジュール表を作ります。これがあると迷いません。
    2. Day1 から順に、講座責任者に「このテーマで本文を書いてください」と依頼します。締め切り日を伝えておきます。
    3. 届いた原稿の誤字や改行を直し、見出しやレイアウトを整えます（小谷の作業）。
    4. 整えた原稿を講座責任者に確認してもらい、OK が出たら次の Day に進みます。
    5. 同じ流れで Day30 まで繰り返します。途中で「どの Day までできたか」を表で管理すると忘れません。
  • メルマガの内容を確認する（講座責任者）
    【ステップ】
    1. Day1 から Day30 まで、全号を順に読みます。
    2. 誤字・脱字がないか、日付や「〇日目」の表記が合っているか、表現で気になる部分がないかをチェックします。メモに書いておきます。
    3. 講座責任者に「ここを直したい」を伝え、直してもらうか一緒に直します。
    4. 直しが終わった号は「確認済み」と印をつけ、全部終わったら講座責任者に「全号確認しました」と報告します。配信前に必ずこの確認をしてください。
  • メルマガの配信準備を終わらせる（単独可）
    【ステップ】
    1. 使う配信ツール（メルマガ配信サービスなど）にログインし、Day1〜30 の原稿を1本ずつ登録します。
    2. 各号の「配信日時」を設定します。Day1 は入会日、Day2 はその翌日…のように、スケジュール通りに設定します。
    3. 必ず「テスト配信」をします。自分のメールアドレスに送って、届くか・文字化けしないか・リンクが動くかを確認します。
    4. 問題がなければ本番の配信設定を有効にします。初めてのときは、技術担当やツールのマニュアルを見ながら進めると安心です。
2.7運営スタッフ（チャット担当者等）4人の採用 一番最優先 初めては難しめ最優先15_スタッフ採用
やさしい言葉で小分け
  • どんな人を探すか決める
    【ステップ】
    1. どんな仕事をしてもらうか、一文で書きます。例：「Discordで会員の質問に丁寧に返信する人」。
    2. 1日 or 1週間にどれくらいの時間働いてほしいか（稼働時間）を決め、書きます。
    3. 経験がなくてもよいか、あるとよいかを決めます。初心者OKなら「未経験可」、そうでなければ「〇〇の経験があると望ましい」と書きます。
    4. この3つをまとめた「探している人のイメージ」をメモしておきます。募集文を書くときに使います。
  • どこで募集するか決める
    【ステップ】
    1. 候補をリストにします。例：求人サイト（ indeed など）・SNS（Twitter/X・Facebook）・知人や仲間に紹介してもらう。
    2. 予算を確認します。求人サイトは掲載料がかかることがあるので、いくらまで出せるか決めます。
    3. いつまでに人を決めたいか（スピード）を考えます。急ぎなら知人紹介やSNSの方が早いことがあります。
    4. 予算とスピードに合う方法を1つか2つに絞り、「ここで募集する」と決めます。
  • どうやって選ぶか決める
    【ステップ】
    1. 選考の流れを決めます。例：応募書類を見る→気になる人に連絡→面接（1回）→採用可否を決める。
    2. 「書類で見るポイント」をメモします。例：丁寧に書けているか、稼働時間が合うか。
    3. 「面接で聞くこと・確認すること」をリストにします。例：自己紹介・今までの経験・いつから・何時間働けるか・質問はあるか。
    4. チェックするポイント（返信の早さ・言葉遣い・相性）を書いておくと、あとで「なぜこの人にしたか」が分かりやすくなります。
  • 求人文を作って掲載する
    【ステップ】
    1. 仕事内容を書きます。何をするか、どのくらいの頻度か、を分かりやすく書きます。
    2. 条件を書きます。稼働時間の目安・報酬・場所（在宅かオフィスか）・期間など。
    3. 応募方法を書きます。「このメールアドレスに送る」「このフォームから送る」など、受け取り方をはっきり書きます。
    4. 書き終えたら、ディレクターまたは有識者に「内容がおかしくないか・法律的に問題ないか」確認してもらいます。
    5. OKをもらったら、決めた募集サイトに投稿します。掲載期間も決めておくとよいです。
  • 応募者を選考する
    【ステップ】
    1. 届いた応募（メール・フォームの回答など）を、一覧になるようにまとめます。名前・連絡先・経歴・希望条件などを表にすると見やすいです。
    2. 決めておいた「書類で見るポイント」に沿って、基準に合う人に印をつけます。
    3. 複数人で選考する場合は、短い評価シート（良い点・気になる点・面接するか）を用意し、同じ基準で見られるようにします。
    4. 面接に呼ぶ人をリストにし、連絡する日時を決めます。呼ばなかった人には、後でお礼と結果を伝えることも忘れずに。
  • 面接をする
    【ステップ】
    1. 面接の日時を決め、相手に連絡します。Zoom のリンクを送るか、対面の場所を伝えます。
    2. 面接で「聞くこと」のリストを用意します。自己紹介・経験・稼働可能時間・質問 など。
    3. 面接のとき、こちらの仕事内容・報酬・ルール・スケジュールを説明し、相手の質問に答えます。
    4. 話した内容はその場でメモを取ります。あとで「誰が何と言っていたか」が分かると、採用を決めるときに役立ちます。
    5. 終わったら「〇日までに結果を連絡します」と伝え、約束した日に連絡します。
  • 採用する人を決める
    【ステップ】
    1. 面接のメモを見直し、採用する人を決めます。目標人数（例：4人）に達するまで、複数回募集することもあります。
    2. 採用する人には「採用のご連絡」を、採用しない人には「今回は見送り」とお礼を添えて連絡します。
    3. 落ちた人への連絡は、できるだけ早く行います。一言でも「ご応募ありがとうございました」と書くと印象がよいです。
    4. 採用した人の名前・連絡先・初日出勤日を一覧にまとめ、次の「説明・案内」に備えます。
  • 採用した人に説明・案内する
    【ステップ】
    1. 採用が決まった人にメールを送ります。挨拶・「初日は〇月〇日です」・何を準備するか を書きます。
    2. 初日の流れを書きます。何時にどこに来るか（or Zoom のリンク）、何をするか、を短く書きます。
    3. 使うツール（Discord など）の名前と、「招待リンクは後で送ります」などと書きます。
    4. 困ったときの連絡先（メール or 電話）を書きます。
    5. 送ったあと、読んだか・質問はないか確認すると、初日のトラブルが減ります。
  • 研修を行う
    【ステップ】
    1. 返信のルールを説明します。何時間以内に返すか、丁寧な言葉遣いにするか、投資の質問は誰に回すか など。
    2. よくある質問と答え（FAQ）を渡し、「同じような質問が来たらこの文を参考に」と伝えます。
    3. 困ったときの連絡先（誰に聞くか・どこに書くか）を伝えます。
    4. 実際のやりとり例（過去の質問と返信）を見せると、イメージがつかみやすくなります。画面共有で一緒に確認するのがおすすめです。
2.2Discord（コミュニティ）の構築最優先05_Discord
やさしい言葉で小分け
  • Discordのサーバー（コミュニティの場）を作る
    【ステップ】
    1. Discord のアプリ（またはブラウザ）を開き、左側の「＋」または「サーバーを追加」をクリックします。
    2. 「自分で作成」を選びます。
    3. サーバー名（例：株コンパスサロン）を入力し、アイコン用の画像があれば設定します。なければ後からでも大丈夫です。
    4. 作成が終わると、自分だけが参加しているサーバーが表示されます。ここが会員の「コミュニティの場」になります。
  • どんなチャンネル（部屋）が必要か決める
    【ステップ】
    1. どんな「部屋」（チャンネル）が必要か、紙やメモに一覧で書きます。
    2. 例：お知らせ（運営からの連絡だけ）・質問（会員が質問を書く）・雑談（自由に話す）・市況まとめ（市況の投稿用）。
    3. 名前を決めたら、次のステップで1つずつ作っていきます。先に一覧を書いておくと、作り忘れがなくなります。
  • チャンネルを作る
    【ステップ】
    1. サーバー名の下、左側のチャンネル一覧の上にある「＋」や「チャンネルを作成」をクリックします。
    2. 「テキストチャンネル」（文字のやりとり用）か「ボイスチャンネル」（音声用）を選びます。まずはテキストで十分です。
    3. チャンネル名を入れます（例：#お知らせ）。説明欄に「運営からのお知らせを貼ります」など短く書いてもよいです。
    4. 決めたチャンネルを、同じ手順で1つずつ作っていきます。
  • 役割（ロール）の名前を決める
    【ステップ】
    1. サーバー名をクリックし、「サーバーの設定」を開きます。
    2. 左のメニューから「ロール」を選び、「ロールを作成」をクリックします。
    3. 役割の名前を決めて追加します。例：会員・運営・講座責任者。1つ作ったら「作成」で保存し、必要ならもう1つ作ります。
    4. あとで「誰にどのチャンネルが見えるか」を決めるときに、このロール名を使います。
  • 誰に何が見えるか・書けるかを決める
    【ステップ】
    1. 各チャンネルの設定（歯車アイコン）を開き、「権限」のところを見ます。
    2. 「会員」ロールには、質問・雑談・市況まとめなど「見て・書いてよい」チャンネルに「メッセージを読む」「メッセージを送信」をオンにします。
    3. 「お知らせ」など運営だけが書くチャンネルは、会員には「メッセージを読む」だけにして、送信は運営ロールだけにします。
    4. プライベートにしたいチャンネルは、「@会員」の権限で「このチャンネルを表示」をオンにし、それ以外のロールでは表示しないようにします。保存を忘れずに。
  • 入会した人が参加する手順を決める
    【ステップ】
    1. 入会した人が Discord に参加するまでの流れを、矢印で書きます。例：決済完了→メールが届く→メールに書いてある招待リンクをクリック→Discord で「参加」を押す。

    3. この流れを1つに決めたら、技術担当に「決済後にこのメールを送る」「会員サイトにこのURLを載せる」と伝え、ずれがないようにします。

    【ステップ】
    1. サーバー名の横の「▼」をクリックし、「招待を作成」を選びます。

    4. リンクは外部に漏れると誰でも参加できてしまうので、会員だけに渡すように注意します。
  • 参加のしかたを案内文に書く
    【ステップ】
    1. 会員向けに「Discord の参加のしかた」を短い文章で書きます。
    2. 手順の例：「①このリンクをクリックする → ②Discord が開く（アプリがなければブラウザで開く）→ ③「参加する」を押す」。
    3. できるだけ、画面のキャプチャ（スクリーンショット）を1〜2枚入れると、初めての人でも迷いにくくなります。
    4. この案内文を、入会メールや会員サイトの「はじめに読んでください」に載せます。
  • ルール（チャンネルごとの注意）を書く
    【ステップ】
    1. 各チャンネルで「書いていいこと」と「してはいけないこと」を短く決めます。
    2. チャンネル名の横の歯車→「トピック」や説明欄に、そのチャンネルのルールを1行で書きます。または「#ルール」というチャンネルを1つ作り、そこに全員向けのルールを書きます。
    3. 例：誹謗中傷禁止・他人の個人情報を書かない・投資の勧誘をしない。分かりやすい言葉で書きます。
    4. 書いたら、講座責任者やディレクターに一度見てもらうと安心です。
  • 動作確認・テストをする
    【ステップ】
    1. 自分とは別の Discord アカウントを用意するか、テスト用のメールで新規登録したアカウントを用意します。
    2. 会員に渡す予定の招待リンクを、そのアカウントで開き、「参加する」を押します。
    3. 会員になったつもりで、チャンネル一覧を見ます。会員用のチャンネルが表示されているか、お知らせだけのチャンネルは読めるか確認します。
    4. 質問用チャンネルにテストで1行送ってみて、送信できるか確認します。問題がなければ本番でも同じ設定で大丈夫です。

2.4配信体制の設計・構築最優先06_配信体制
やさしい言葉で小分け
  • いつ配信するか決める
    【ステップ】
    1. 配信の「頻度」を決めます。毎週〇曜日か、毎日か、週2回か など。
    2. 何時頃に送るかも決めます。会員が読みやすい時間帯（例：朝8時・夜6時）にするとよいです。
    3. 決めたら、カレンダーやスプレッドシートに「〇月〇日（〇）配信」と書いておきます。忘れ防止になります。
    4. 担当者が変わる場合も、このカレンダーを見れば誰がいつ送るか分かるようにしておきます。
  • どんな方法で配信するか決める
    【ステップ】
    1. 配信する内容を「どこで」届けるか決めます。DiscordとLINEを使います。
    2. 重要なお知らせはDiscord＋LINEの2つで届けると見逃しが減ります。
    3. 「通常のお知らせはDiscord」「緊急はDiscord＋LINE」のように、種類ごとに届け先を決めておくと、初めての人でも迷いません。
    4. 決めた方法をメモし、手順書に書きます。
  • 原稿の作り方の流れを決める
    【ステップ】
    1. 原稿を「誰が」書くか決めます。講座責任者か、小谷か、外注か。
    2. 書いた原稿を「誰が」確認するか決めます。確認者がOKを出したら次に進むようにします。
    3. 「いつまでに」原稿を提出するか、「いつまでに」OKをもらうかを決め、配信日の前から逆算してスケジュールを書きます。
    4. この流れを1本の図や文章にし、関係者で共有します。流れが1つに決まっていると、忘れや遅れが減ります。
  • 誰が確認・OKするか決める
    【ステップ】
    1. 配信前に「内容でOK」を出す人を1人決めます。講座責任者か、ディレクターか、など。
    2. 「OKが出るまで配信しない」というルールを共有します。OKを出す人と、配信する人が違う場合、連絡の取り方（メールで「OK」と返すなど）も決めます。
    3. 決めた人の名前と、連絡方法を手順書に書いておきます。初めての人でも「誰に確認すればよいか」が分かります。
  • 実際に配信する手順を決める
    【ステップ】
    1. 実際に配信するときの操作を、順番に書きます。例：①ログインする ②「新規配信」をクリック ③原稿を貼る ④プレビューで表示を確認 ⑤送信ボタンを押す。
    2. 使うツールの画面に合わせて、「どこをクリックするか」を書きます。スクショがあると分かりやすいです。
    3. 送信前には必ず「テスト送信」で自分のアドレスに送り、届くか・表示がおかしくないかを確認する、と書いておきます。
    4. この手順を、初めてやる人が1人でできるように文章化します。
  • 配信の手順書を書く
    【ステップ】
    1. ここまでで決めた「いつ・何を・誰が・どのツールで」を、1本の手順書にまとめます。
    2. 見出しをつけて、初めての人でも上から順に読めばできるようにします。例：配信の流れ・原稿の依頼の仕方・確認の取り方・実際の送信手順。
    3. 使うツールの名前・ログインURL・担当者を書いておきます。
    4. 手順書を関係者に共有し、「分かりにくいところはないか」聞いて直します。
  • 配信前チェックリストを作る
    【ステップ】
    1. 配信前に必ず確認する項目を、チェックリストにします。
    2. 例：配信日付は合っているか・誤字・脱字はないか・宛先（送り先）は正しいか・リンクは正しく飛ぶか・画像は表示されるか。
    3. このリストを印刷するか、手順書の最後に載せて、配信のたびに1つずつチェックするようにします。
    4. チェックした人・日付をメモしておくと、あとで「確認したか」が分かります。

    2. 改行のルールを決めます。見出しのあとは1行空ける、段落のあとは1行空ける、など。
    3. 書いてはいけない表現を決めます。投資の勧誘や断定表現など、講座責任者とリストにします。
    4. この書式をテンプレの上か、手順書に書いておき、書く人全員が守るようにします。

2.6Discord内での質問対応の基本体制最優先08_質問対応
やさしい言葉で小分け
  • 質問をどこで受け取るか決める
    【ステップ】
    1. 会員が質問を送る場所を決めます。Discordの#質問チャンネルを使います。
    2. 場所は1つに絞ります。たくさんあると「どこに書けばよいか」と会員が迷い、対応する側も見落としが増えます。
    3. 決めた場所を会員向けガイドと手順書に書きます。「質問は #質問 チャンネルに書いてください」のように。
  • 回答の流れを決める
    【ステップ】
    1. 質問が届いたら「誰が」まず受け取るか決めます。当番制か、全員で見るか。
    2. 誰が答えるか振る（割り振る）ルールを決めます。当番の人が答えるか、内容によって講座責任者に回すか。
    3. 返信したあと、記録を残すか決めます。同じ質問が来たときに参考になるので、簡単でよいので「どんな質問・どう答えたか」をメモしておくとよいです。
    4. この流れを「受け取る→振る→返信→記録」と図にして、担当者で共有します。
  • 何時間以内に返すかなど、基準を決める
    【ステップ】
    1. 何時間以内に初回の返信をするか決めます。例：24時間以内。会員に「〇時間以内を目安に返信します」と伝えてもよいです。
    2. 投資の内容（銘柄・売買の判断など）の質問は、スタッフではなく講座責任者に回す、と決めます。
    3. 答えてよい範囲と、答えてはいけないこと（個別銘柄の推奨など）をリストにします。
    4. この基準を手順書に書いておくと、初めて対応する人でも迷いません。
  • 質問対応の手順書を書く
    【ステップ】
    1. 質問をどこで見るか（チャンネルを開く・通知の確認）を書きます。
    2. 返信の書き方のルールを書きます。挨拶を入れる・丁寧語を使う・結論を先に書く、など。
    3. 「このような質問は講座責任者に回す」という判断基準を書きます。例：銘柄の買い時・売り時についての質問。
    4. 回すときは、誰に・どの手段で（メール・チャット）伝えるかも書きます。この手順書をスタッフ全員が読めるようにします。
  • よくある質問と答え（FAQ）を作る
    【ステップ】
    1. 過去にあった質問や、ありそうな質問をリストにします。
    2. それぞれに「答え」を書き、Q&A形式で一覧にします。例：Q. ログインできません → A. まずパスワードをリセットしてみてください。URLは〇〇です。
    3. 講座責任者や技術担当に、答えの内容が正しいか確認してもらいます。
    4. このFAQをスタッフ用の手順書に載せ、会員向けにも「よくある質問」として載せると便利です。
  • よくある質問用の返信文を作る
    【ステップ】
    1. FAQの各質問に対して、「そのままコピーして送れる」返信文を用意します。
    2. 挨拶＋答え＋「他に質問があればお知らせください」のように、短く完成した文にします。
    3. スタッフが「この質問にはこの返信文」と選べるように、FAQと対応表にしておきます。
    4. 返信文はときどき見直し、内容が古くなっていないか確認します。
  • 誰が対応するか決める

    2. それらを何に使うか（サービス提供・お知らせ・問い合わせ対応）を書きます。
    3. 保管の方法・第三者に提供するか（提供する場合はどこに・何のためか）を書きます。
    4. ドラフトをディレクターまたは有識者にチェックしてもらい、OKをもらってから公開します。
  • 個人情報の扱いを書く（ディレクター or 有識者にチェック）
    【ステップ】
    1. 個人情報をいつ・どのように取得するか、利用目的は何かを簡潔に書きます。
    2. 保管期間（いつまで持つか）と、開示・訂正・削除の請求があったときの対応を書きます。
    3. プライバシーポリシーと重複しないよう、必要なら「詳細はプライバシーポリシーへ」と誘導します。
    4. ディレクターまたは有識者にチェックしてもらい、問題なければ載せます。
  • コミュニティのルールを書く（ドラフト→ディレクターにチェック）
    【ステップ】
    1. コミュニティで守ってほしいルールをリストにします。例：誹謗中傷禁止・他者の個人情報や発言を無断で外に出さない・投資の勧誘をしない。
    2. 短く、読みやすい文にします。長いと読まれません。
    3. ドラフトをディレクターにチェックしてもらいます。
    4. このルールをDiscordの「#ルール」チャンネルに載せます。
  • Discordのチャンネルルールを書く（ドラフト→ディレクターにチェック）
    【ステップ】
    1. Discord の各チャンネルで「このチャンネルでは何を書いてよいか・NGは何か」を決めます。
    2. チャンネル設定の「トピック」や説明欄に1行で書くか、「#ルール」チャンネルを作って一覧で書きます。
    3. 例：#質問 → サロン・教材についての質問のみ。銘柄の推奨は書かない。
    4. ドラフトをディレクターにチェックしてもらい、問題なければ反映します。
  • ガイドラインを書く（ドラフト→ディレクター or 有識者にチェック）
    【ステップ】
    1. 運営側が「どんなときにどう対応するか」の判断基準をまとめます。例：ルール違反への警告・繰り返し違反した場合の対応・問い合わせへの返信方針。
    2. 対応方針を短い文章で書き、スタッフ全員が同じ基準で動けるようにします。
    3. ドラフトをディレクターまたは有識者に確認してもらいます。
    4. ガイドラインは主に運営用ですが、会員に公開する部分（ルールの理由など）があれば分けて書きます。
  • Discordに載せる（単独可）
    【ステップ】
    1. Discordの#ルールチャンネルに「利用規約・プライバシーポリシー・免責事項」の内容を投稿します。
    2. 長い場合は、Googleドキュメントなどにまとめてリンクを貼る方法もあります。
    3. 投稿したらピン留めして、いつでも見られるようにします。
    4. 会員として確認し、正しく表示されているか確認します。
  • 入会時に案内する（単独可）
    【ステップ】
    1. Discordのウェルカムメッセージや#ルールチャンネルで、規約・ポリシーの場所を案内します。
    2. 「サロンをご利用の前に、#ルールをご確認ください」と書きます。
    3. リンクをクリックして、正しい場所に飛ぶか確認します。
    4. これで会員が入会時に規約を確認できる状態になります。
2.12オフ会・交流イベント（1ヶ月以内は不要）低優先—
やさしい言葉で小分け
  • イベントの内容を決める
    【ステップ】
    1. イベントの「種類」を決めます。交流会（顔を合わせて話す）・勉強会（テーマを学ぶ）・懇親会（食事しながら話す）など。
    2. 会員同士が話せる場にするか、運営が進行する形式にするかを決めます。
    3. 初めてのときは、形を1つに絞ると準備がしやすいです。例：まずは少人数の交流会から。
  • 日時・場所を決める
    【ステップ】
    1. 日時は会員が集まりやすい時間にします。土日や平日の夜が選ばれやすいです。
    2. 場所を決めます。会議室・カフェ・オンライン（Zoomなど）のどれにするか。予算と人数で決めます。
    3. 候補の日時・場所を2〜3パターン挙げ、関係者と「この日時・場所でよいか」確認してから確定します。
  • 参加者を募る
    【ステップ】
    1. 参加者をどこで募るか決めます。Discordの#運営からのお知らせとLINEで募集します。
    2. 案内文に「〇月〇日オフ会をやります。参加したい方は〇〇までに〇〇で申し込みください」と書きます。
    3. 申し込み方法をはっきり書きます。Discordで反応するか、フォームで受け付けるか。
    4. 締め切り日も書いておくと、人数が把握しやすくなります。
  • 会場を押さえる
    【ステップ】
    1. 申し込み人数（または予想人数）を確認し、それに合った広さの会場を探します。
    2. 会議室やカフェ・レンタルスペースを予約します。料金・キャンセル条件を確認してから契約します。
    3. 予約が取れたら、住所・アクセス・開場時間をメモし、参加者に後で案内できるようにします。
  • 告知・募集をする
    【ステップ】
    1. 告知文に「日時・場所・定員・申し込み締め切り・持参するもの」を書きます。
    2. 会員が参加しやすいよう、DiscordとLINEの両方で送るとよいです。
    3. 送った日をメモし、締め切り前にリマインド（もう一度案内）するか決めておきます。
    4. 申し込みが少ない場合の対応（延期するか・少人数でやるか）も、事前に決めておくと安心です。
  • 当日の準備をする
    【ステップ】
    1. 受付の仕方を決めます。入口で名前を確認する・名札を渡す・参加者リストにチェックする、など。
    2. 名札・アンケート用紙・筆記具を用意します。少人数なら簡易的で大丈夫です。
    3. 当日の流れ（何時に開始・何時頃に終了）をメモし、進行役が分かるようにします。
    4. 会場の設営（椅子・テーブル・看板）が必要なら、開始時刻の何分前に入るか決めておきます。
  • イベントを実施する
    【ステップ】
    1. 開始時刻になったら、進行役が挨拶をしてイベントを始めます。
    2. 進行は事前に決めた流れに沿って進めます。時間が押したら、どこを短くするかも決めておくとよいです。
    3. 写真や動画を撮る場合は、事前に「SNSに載せてもよいか」参加者に許可を取っておきます。
    4. 終了時刻が近づいたら、挨拶と次回の案内をして締めます。
  • 終わったら振り返り・改善する
    【ステップ】
    1. 終了後、参加者からもらったアンケートや声を読み、良い点・改善点をメモします。
    2. 運営側でも「うまくいったこと・難しかったこと・次に直したいこと」を短くまとめます。
    3. このメモを保管し、次回のイベントを計画するときに見返します。
    4. 同じ失敗を防ぎ、参加者が楽しめるイベントに近づけていきます。
2.13詳細なKPI分析・レポート（1ヶ月以降）低優先—
やさしい言葉で小分け
  • 何を分析するか決める
    【ステップ】
    1. 分析したい「数字」をリストにします。例：入会数・離脱（退会）率・動画の視聴率・Discordの投稿数・メルマガの開封率。
    2. 全部は追いきれないので、見たい数字を3〜5個に絞ります。
    3. 絞った項目をディレクターや関係者と共有し、「この数字を見れば運営の様子が分かる」ようにします。
  • 分析のやり方を決める
    【ステップ】
    1. 各数字を「どこから」取るか決めます。決済システムの管理画面・メルマガ配信ツール・Discordの統計など。
    2. データを「いつ」集めるか決めます。月1回まとめるか、週1回か。
    3. 集める人（誰がデータを取るか）と、取り方をメモします。初めてのときは技術担当に教えてもらうとよいです。
  • レポートの形を決める
    【ステップ】
    1. レポートの「形」を決めます。1枚のサマリー（要約）にするか、数ページに分けるか。
    2. グラフを入れるか決めます。数字だけより、グラフがあると傾向が分かりやすくなります。
    3. 見出しの並び（数値一覧・前月比・所感・次のアクション）を決めておくと、毎回同じ形で書けます。
    4. この形をテンプレにして、毎回コピーして使うと楽です。
  • データを分析する
    【ステップ】
    1. 決めた方法で、各数字を集めます。各ツールの画面からコピーするか、CSVでダウンロードします。
    2. 前月と比べて「増えたか・減ったか」をメモします。
    3. 傾向を短く書きます。例：「入会数は先月より〇人増。Discordの参加率は横ばい」。
    4. 分からない数字があれば、技術担当に聞いておきます。
  • レポートを書く
    【ステップ】
    1. 集めた数字と前月比・傾向を、レポートの形にまとめて書きます。
    2. 「気づいたこと」を短く書きます。例：〇〇が減っているので、次月は〇〇を試したい。
    3. 「次のアクション」を1〜3個書きます。何をいつまでにするか。
    4. レポートをディレクターに提出し、フィードバックがあれば次回に活かします。
  • 改善案を出す
    【ステップ】
    1. レポートの数字を見て、「ここを直すとよさそう」という改善案を1〜3個考えます。
    2. 例：離脱率が高い→入会直後のフォローを増やす。動画視聴率が低い→タイトルや案内を分かりやすくする。
    3. 改善案を短い文章で書き、ディレクターに提案します。
    4. 採用された案は、次のレポートで「やってみたこと・結果」として書くと、改善の流れが残ります。
2.14高度なエンゲージメント施策（会員が増えてから）低優先—
やさしい言葉で小分け
  • どんな施策をするか考える
    【ステップ】
    1. 会員が「参加したくなる」施策をリストにします。例：アンケート・限定ライブ配信・〇日間チャレンジ企画・読書会・テーマ別ディスカッション。
    2. 自分たちでできそうなものと、予算・人手がかかるものを分けておきます。
    3. リストを関係者と共有し、「まずはどれからやるか」を話し合います。
  • 施策の内容を決める
    【ステップ】
    1. 施策を1つ選び、内容を決めます。何をするか・誰が担当するか・いつまでにやるか。
    2. 担当者と「やることリスト」を共有し、役割がぶれないようにします。
    3. 途中で変更があれば、関係者にすぐ共有します。
  • いつ何をするか計画を書く
    【ステップ】
    1. カレンダーや表（スプレッドシート）を用意し、いつ何をするか書きます。
    2. 告知する日・参加申し込みの締め切り・実施日・結果共有の日、を入れます。
    3. この計画を関係者で共有し、忘れないようにします。
  • 施策を実行する
    【ステップ】
    1. 告知を出します。Discord・LINEで、日時・内容・申し込み方法を書きます。
    2. 参加申し込みを受け付け、人数を把握します。
    3. 実施日になったら、計画どおりに進めます。
    4. 終わったら参加者にお礼を伝え、結果や写真・感想を共有すると、次回も参加しやすくなります。
  • 効果を測る
    【ステップ】
    1. 施策の「効果」をどう測るか決めます。参加人数・アンケートの感想・Discordの投稿数 など。
    2. 終了後に、決めた数字や感想を記録します。表やメモに残しておきます。
    3. 記録は次回の改善やレポートに使えるので、簡易でよいので残すようにします。
  • 結果を見て直す・よくする
    【ステップ】
    1. 施策が終わったあと、「うまくいった点」と「いかなかった点」をメモします。
    2. 参加者の声や数字を見て、何が良かったか・何を直すか考えます。
    3. このメモを保管し、次に同じような施策をするときに見返します。
    4. 改善を続けると、会員の満足度や参加率が上がりやすくなります。
